# Yura Inner State Visualizer

ゆらの感情・Drive・Activity状態を、光の粒子の色、密度、動き、形状として表示するサイドカーです。
`ai-liver-yura` モノレポの `gui/` 配下で、本体とは別プロセスとして動作します。

## 画面コンセプト

画面名は **「こころの潮流」** とします。
会話画面の「ことばの水面」、配信管理画面の「配信の灯台」と同じく、海を用いた抽象的な命名で統一します。

- ヘッダーには固定の画面名を表示します。
- 現在の感情表現はヘッダーのタイトルとして使用せず、画面下部の `MOOD` に表示します。
- 現在のActivityは画面下部の `NOW` に表示します。

## 起動

```bash
cd ai-liver-yura/gui/yura-inner-state-visualizer
python3 server.py
```

ブラウザで <http://127.0.0.1:8765> を開き、その後ゆらを起動します。

```bash
cd ai-liver-yura
.venv/bin/python -m app
```

ゆらは`127.0.0.1:8766/UDP`へ状態を配信します。ビジュアライザーが起動していなくても、ゆらの動作には影響しません。

## 通信断の判定と画面遷移

- 最後の状態スナップショット受信から **45秒** 更新がない場合に、状態ソースを利用不可として扱います。
- SSE接続自体が切断された場合は、45秒を待たず再接続状態へ移行します。
- 通信断時は光の粒子を即座に消さず、十数秒かけて緩やかに散逸させます。
- 状態受信が復帰した場合も、光の粒子を数秒かけて中心へ集めます。
- 通信断・復帰アニメーションは、状態値の意味ではなく接続状態を表す演出です。

## 単独確認

別のターミナルで次を実行すると、感情状態を模擬できます。

```bash
python3 simulator.py
```

## 表現の対応

- valence: 寒色から暖色への色相
- mood: 基調色と画面下部の感情名
- arousal: 粒子の速度、脈動、発光
- talkativeness: 表面の流動性
- curiosity: 全体の広がり
- engagement: 粒子の凝集度
- boredom: 形状の扁平化と散逸
- energy: 明るさと粒径

通信には入力本文、会話履歴、ウィンドウ名、キー入力を含めません。将来のPC観測機能とは別の読み取り専用境界です。
