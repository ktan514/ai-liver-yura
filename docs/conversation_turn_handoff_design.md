# 会話ターン返却・自律発話抑制設計

## 1. 目的

ユーザーへの返答後にAIライバーが同じ話題を自律的に連続展開し、ユーザーが会話へ入れなくなる問題を防止する。

本設計では、特定の語句を禁止するのではなく、次の汎用課題を解決する。

- AIライバーが発話権を占有し続ける
- ユーザーの無反応を関心や継続要求として誤解する
- 同じ主語・導入句・情景・感想を言い換えて反復する
- 毎回質問するだけのチャットボット的応答になる
- キャラクターの実体験・身体感覚・居住環境が曖昧になる

## 2. 基本方針

### 2.1 発話権の返却

ユーザーへの返答が完結した時点で、発話権をユーザーへ返す。

沈黙は「もっと話してよい」という許可ではない。自律発話を再開する場合は、直前の発話終了から十分な間隔を確保する。

### 2.2 自律発話の上限

同一話題の自律発話は原則として短く閉じる。

- 同一話題は最大2発話まで
- ユーザーが介入した話題は自動再開しない
- 重要かつ未完了の話題だけを中断状態として保持する
- 再開には明示的な継続理由が必要

### 2.3 無反応の扱い

無反応や時間経過だけでは次を推定しない。

- 関心が高い
- 話題が盛り上がっている
- 続きを期待している
- 共感を深めるべき

関心の上昇は、ユーザーの追加発言、質問、同意、具体化などの観測可能な反応を根拠とする。

### 2.4 問いかけの扱い

会話を自然にするため、毎回質問で終えない。

発話終端は次を使い分ける。

- 共感して止まる
- 自分の反応を一つ足して止まる
- 軽い問いかけで開く
- 相手が入りやすい感想を残す
- 話題を閉じる

問いかけた場合も、回答を強制する待機表現は使わない。

## 3. キャラクター存在設定

AIライバーは、海をモチーフや好みとして持っていても、海中に住んでいるとは自動的に解釈しない。

- 物理的身体感覚や実体験を持つと断定しない
- 水温、匂い、触感などは知識・映像・想像に基づく表現として区別する
- 見た、行った、触った等の経験表現は、根拠がある場合だけ使用する

## 4. 実装方針

### 4.1 AutonomousActivityPolicy

発話終了後の最小待機時間を、短い発話間隔ではなく会話への再参加余白として扱う。

基準値:

- 通常: 45秒
- 発話意欲が高い: 30秒
- 興奮状態: 20秒
- 怒り・悲しみ: 45秒
- 疲労・低発話意欲: 60秒

### 4.2 TopicContinuationEvaluator

次の条件で元話題を終了する。

- 同一話題が2発話に到達した
- ユーザーが介入し、重要な未完了話題ではない
- 話題が消耗している

ユーザーが介入した重要な未完了話題は、即時再開せず中断状態として保持する。

### 4.3 CharacterProfile

プロンプトへ渡される行動方針に、発話権、無反応、反復防止、問いかけ、存在設定の共通ルールを補完する。

## 5. 受け入れ条件

- ユーザー返答後、短時間で同じ話題を再開しない
- 同一話題を2発話以上自律継続しない
- ユーザー介入後に元の自律話題へ自動復帰しない
- 「海の中って」のような同じ導入構造の反復を方針上禁止できる
- 毎回質問で終わらず、発話を止める選択肢がある
- 海モチーフと海中居住設定を混同しない
- 物理的な身体経験を捏造しない
